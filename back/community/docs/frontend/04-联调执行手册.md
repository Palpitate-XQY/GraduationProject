# 04-联调执行手册

## 1. 目标与范围
- 目标：确保管理端页面设计可直接驱动前后端联调，主链路一次跑通。
- 范围：仅使用当前后端已实现接口，不新增后端模块。
- 存储策略：优先 `LOCAL`，七牛云作为配置能力保留。

## 2. 联调准备

### 2.1 环境准备
- 后端服务：`back/community` 启动成功。
- 数据库：执行 `sql/01_schema.sql`、`sql/02_index.sql`、`sql/03_init_data.sql`。
- Redis：本地可用。
- 接口文档：
  - `http://127.0.0.1:8080/swagger-ui/index.html`
  - `http://127.0.0.1:8080/doc.html`

### 2.2 账号与基础变量

- 组织样例：

## 3. 固定联调顺序（前端执行脚本）

### Step 1 数据初始化确认
1. 执行 SQL 三件套。
2. 管理端登录后查询：
   - 组织树 `GET /api/org/tree`
   - 角色列表 `GET /api/system/roles`
3. 验证样例组织与基础角色存在。

### Step 2 认证链路
1. 居民注册：`POST /api/auth/resident/register`
2. 发送重置验证码：`POST /api/auth/password/reset-code`
3. 验证码重置：`POST /api/auth/password/reset-by-code`
4. 居民登录与管理员登录：`POST /api/auth/login`
5. 获取当前用户：`GET /api/auth/me`

### Step 3 存储链路
1. 管理员设置本地存储：`PUT /api/system/storage-config`
2. 上传公告附件：`POST /api/files/upload`
3. 上传活动附件：`POST /api/files/upload`
4. 上传报修图片：`POST /api/files/upload`
5. 校验文件详情和下载：`GET /api/files/{id}`、`GET /api/files/{id}/download`

### Step 4 公告链路
1. 新建 Markdown 公告并绑定附件：`POST /api/notices`
2. 公告发布：`POST /api/notices/{id}/publish`
3. 管理端详情查看：`GET /api/notices/{id}`
4. 居民端可见性验证：
   - `GET /api/notices/resident/page`
   - `GET /api/notices/resident/{id}`
5. 验证 `contentHtml` 非空，`attachments` 数组正确。

### Step 5 活动链路
1. 新建 Markdown 活动并绑定附件：`POST /api/activities`
2. 发布活动：`POST /api/activities/{id}/publish`
3. 居民报名：`POST /api/activities/{id}/signup`
4. 居民取消报名：`POST /api/activities/{id}/cancel-signup`
5. 管理端查询：
   - 报名名单 `GET /api/activities/{id}/signups`
   - 统计 `GET /api/activities/{id}/stats`

### Step 6 报修链路
1. 居民创建报修（含附件）：`POST /api/repairs`
2. 物业受理：`POST /api/repairs/{id}/accept`
3. 物业分派：`POST /api/repairs/{id}/assign`
4. 维修员接单：`POST /api/repairs/{id}/take`
5. 维修提交结果：`POST /api/repairs/{id}/submit`
6. 居民确认解决：`POST /api/repairs/{id}/confirm`
7. 居民评价：`POST /api/repairs/{id}/evaluate`
8. 校验流转日志：`GET /api/repairs/{id}/logs`

### Step 7 系统链路
1. 创建物业管理员、维修员账号：
   - `POST /api/system/users`
2. 角色分配：
   - `PUT /api/system/users/roles`
3. 越权拦截验证：
   - 使用居民 token 访问 `GET /api/system/users`，预期 `code=403`
4. 日志查询验证：
   - `GET /api/logs/logins`
   - `GET /api/logs/operations`

## 4. 管理端页面联调清单
| 模块 | 页面 | 联调动作 | 通过标准 |
|---|---|---|---|
| 认证 | 登录页 | 登录 -> 拉取 me -> 构建菜单 | 菜单仅显示有权限项 |
| 公告 | 列表/编辑/详情 | 新建+发布+详情渲染 | `contentHtml` 正常渲染，附件可下载 |
| 活动 | 列表/编辑/报名统计 | 新建+发布+报名+取消 | 报名统计同步更新 |
| 报修 | 列表/详情/流转日志 | 受理->分派->接单->提交 | 状态流转合法，无 1005 |
| 系统管理 | 用户/角色/权限 | 新增账号+分配角色 | 权限即时生效 |
| 组织 | 组织树/关联 | 新增组织+维护关联 | 树与关联数据一致 |
| 日志 | 登录/操作日志 | 条件查询 | 返回分页数据 |
| 存储配置 | 配置页 | 切换 LOCAL 并保存 | 上传下载路径可用 |

## 5. 关键断言与失败处理
- 权限不越权：居民访问系统管理接口返回 `403`。
- 数据范围不串区：社区/物业账号只能看到授权范围数据。
- 附件可访问：文件详情可查，下载成功。
- 状态机安全：报修非法跳转返回 `1005`。
- Markdown 渲染：公告/活动详情返回 `contentHtml`。

失败时处理规则：
1. 先保留失败请求与 `traceId`。
2. 在后端日志定位对应异常。
3. 修复后只回归失败链路与其前置链路，不全量重跑。
4. 将修复结论写入 `docs/09-测试方案与测试记录.md`。
