# 02-管理端页面功能规格

## 1. 全局交互规范
- 分页请求参数统一：`current`、`size`；默认 `1/10`，最大 `100`。
- 查询页统一支持“重置条件”，重置后回到第一页。
- 列表刷新后保留当前筛选条件，不清空用户输入。
- 业务错误提示统一展示后端 `message`，并保留 `traceId` 供排障。
- 权限不足（`code=403`）时按钮直接隐藏；路由拦截后跳转 `403` 页面。
- 登录失效（`code=401`）时清空本地令牌并跳转登录页。

## 2. 认证模块

### 2.1 登录页
- 路由：`/login`
- 接口：`POST /api/auth/login`
- 表单字段：

| 字段 | 必填 | 校验规则 | 说明 |
|---|---|---|---|
| `username` | 是 | 非空 | 用户名 |
| `password` | 是 | 非空 | 密码 |

- 成功行为：
  - 保存 `accessToken`。
  - 立即调用 `GET /api/auth/me` 拉取权限码集合并构建动态菜单。
  - 若 `mustChangePassword=1`，强制跳转修改密码弹窗。
- 异常态：
  - `1001`：提示“用户名或密码错误”。
  - `1002`：提示“用户已禁用”。
  - 其他业务码：透传 `message`。

### 2.2 修改密码页
- 路由：`/profile/password`
- 接口：`PUT /api/auth/password`
- 表单字段：

| 字段 | 必填 | 校验规则 |
|---|---|---|
| `oldPassword` | 是 | 非空 |
| `newPassword` | 是 | 长度 8-64 |

- 交互规则：
  - 成功后提示并要求重新登录。
  - 失败时保留输入框，密码字段不回显。

### 2.3 忘记密码页（管理端排障入口）
- 路由：`/auth/forgot-password`
- 接口：
  - `POST /api/auth/password/reset-code`
  - `POST /api/auth/password/reset-by-code`
- 字段与校验：
  - `username`：非空
  - `phone`：`^1\\d{10}$`
  - `code`：6位数字
  - `newPassword`：长度 8-64

## 3. 公告模块

### 3.1 公告列表页
- 路由：`/notice/list`
- 页面权限：`notice:list`
- 查询条件：

| 字段 | 类型 | 说明 |
|---|---|---|
| `keyword` | string | 标题模糊查询 |
| `noticeType` | enum | `STREET_COMMUNITY`、`PROPERTY` |
| `status` | enum | `DRAFT`、`PUBLISHED`、`RECALLED` |
| `current/size` | number | 分页 |

- 表格列：
  - `id`、`title`、`noticeType`、`status`、`topFlag`、`publisherOrgId`、`publishTime`、`createTime`
- 操作按钮：
  - 新建（`notice:create`）
  - 编辑（`notice:update`）
  - 删除（`notice:delete`）
  - 发布（`notice:publish`，仅草稿）
  - 撤回（`notice:recall`，仅已发布）
  - 详情（`notice:view`）

### 3.2 公告新建/编辑页
- 路由：
  - 新建 `/notice/create`
  - 编辑 `/notice/edit/:id`
- 接口：
  - `POST /api/notices`
  - `PUT /api/notices`
  - `GET /api/notices/{id}`（编辑回填）
- 表单字段：

| 字段 | 必填 | 校验 | 说明 |
|---|---|---|---|
| `noticeType` | 是 | 非空 | 公告类型 |
| `title` | 是 | 非空 | 标题 |
| `content` | 是 | 非空 | Markdown 正文 |
| `coverFileId` | 否 | 数值 | 封面文件ID |
| `attachmentJson` | 否 | JSON字符串 | 附件引用 |
| `topFlag` | 否 | 0/1 | 是否置顶 |
| `scopeItems` | 是 | 非空数组 | 可见范围 |

- Markdown 交互：
  - 左侧编辑器写 `content`，右侧实时预览。
  - 提交前仅发 Markdown 原文，不发 HTML。
- 附件交互：
  - 通过文件上传组件先上传，得到 `fileId`。
  - 前端维护结构化附件数组，再序列化为 `attachmentJson` 提交。

### 3.3 公告详情页
- 路由：`/notice/detail/:id`
- 接口：`GET /api/notices/{id}`
- 展示字段：
  - `contentHtml` 直接渲染
  - `attachments` 渲染附件卡片（名称、大小、下载）
  - 保留 `content` 原文用于“查看源码”

### 3.4 空态与异常态
- 列表空态：展示“暂无公告”，提供“新建公告”入口（有权限时）。
- 发布失败：保留当前数据并提示失败原因，不自动刷新页面。

## 4. 活动模块

### 4.1 活动列表页
- 路由：`/activity/list`
- 页面权限：`activity:list`
- 查询条件：`keyword`、`status`、`current`、`size`
- 表格列：
  - `id`、`title`、`status`、`activityStartTime`、`activityEndTime`、`signupStartTime`、`signupEndTime`、`maxParticipants`、`publishTime`
- 操作：
  - 新建/编辑/删除/发布/撤回/详情
  - 报名名单（`activity:signup:list`）
  - 统计（`activity:stats`）

### 4.2 活动新建/编辑页
- 路由：
  - 新建 `/activity/create`
  - 编辑 `/activity/edit/:id`
- 接口：
  - `POST /api/activities`
  - `PUT /api/activities`
  - `GET /api/activities/{id}`
- 表单字段：

| 字段 | 必填 | 校验 | 说明 |
|---|---|---|---|
| `title` | 是 | 非空 | 活动标题 |
| `content` | 是 | 非空 | Markdown 正文 |
| `coverFileId` | 否 | 数值 | 封面 |
| `attachmentJson` | 否 | JSON字符串 | 附件 |
| `activityStartTime` | 是 | 非空 | 活动开始 |
| `activityEndTime` | 是 | 非空 | 活动结束 |
| `signupStartTime` | 是 | 非空 | 报名开始 |
| `signupEndTime` | 是 | 非空 | 报名截止 |
| `location` | 是 | 非空 | 地点 |
| `maxParticipants` | 是 | 正整数 | 人数上限 |
| `scopeItems` | 是 | 非空数组 | 可见范围 |

- 前端补充校验：
  - `activityEndTime > activityStartTime`
  - `signupEndTime >= signupStartTime`
  - `signupEndTime <= activityStartTime`

### 4.3 活动详情与报名统计页
- 路由：
  - `/activity/detail/:id`
  - `/activity/signups/:id`
- 接口：
  - `GET /api/activities/{id}`
  - `GET /api/activities/{id}/signups`
  - `GET /api/activities/{id}/stats`
- 展示重点：
  - `contentHtml`、附件列表
  - 报名人数、余量、状态分布

### 4.4 空态与异常态
- 活动列表空态：提示“暂无活动”，引导新建。
- 发布失败：提示并保留草稿，避免丢失输入。

## 5. 报修模块

### 5.1 工单管理列表页
- 路由：`/repair/manage`
- 页面权限：`repair:manage:list`
- 查询条件：

| 字段 | 说明 |
|---|---|
| `keyword` | 标题/单号模糊查询 |
| `status` | 工单状态 |
| `complexOrgId` | 所属小区 |
| `propertyCompanyOrgId` | 物业公司 |
| `maintainerUserId` | 维修员 |
| `residentUserId` | 居民 |
| `createStartTime/createEndTime` | 创建时间范围 |
| `current/size` | 分页 |

- 表格列：
  - `orderNo`、`title`、`status`、`emergencyLevel`、`complexOrgId`、`propertyCompanyOrgId`、`maintainerUserId`、`urgeCount`、`createTime`
- 操作：
  - 详情（`repair:manage:view`）

### 5.2 工单详情与流转操作页
- 路由：`/repair/detail/:id`
- 页面权限：`repair:manage:view`
- 详情接口：`GET /api/repairs/{id}`
- 动态操作按钮与表单：

| 操作 | 权限 | 接口 | 表单字段 | 可执行状态 |
|---|---|---|---|---|
| 受理 | `repair:accept` | `POST /api/repairs/{id}/accept` | `remark` | `PENDING_ACCEPT` |
| 驳回 | `repair:reject` | `POST /api/repairs/{id}/reject` | `reason`(必填) | `PENDING_ACCEPT` |
| 分派 | `repair:assign` | `POST /api/repairs/{id}/assign` | `maintainerUserId`(必填), `remark` | `ACCEPTED/REOPENED` |
| 接单 | `repair:take` | `POST /api/repairs/{id}/take` | `remark` | `ASSIGNED` |
| 处理 | `repair:process` | `POST /api/repairs/{id}/process` | `processDesc`(必填), `attachmentFileIds` | `PROCESSING` |
| 提交结果 | `repair:submit` | `POST /api/repairs/{id}/submit` | `finishDesc`(必填), `attachmentFileIds` | `PROCESSING` |
| 关闭 | `repair:close` | `POST /api/repairs/{id}/close` | `remark` | `REJECTED/REOPENED` |

- 说明：
  - 前端按“权限 + 状态”双条件显示按钮，后端仍是最终校验。
  - 附件上传统一复用文件上传组件，提交 `attachmentFileIds`。

### 5.3 工单流转日志页
- 路由：`/repair/logs/:id`
- 权限：`repair:log:view`
- 接口：`GET /api/repairs/{id}/logs`
- 展示字段：`fromStatus`、`toStatus`、`operationType`、`operatorUserId`、`operationRemark`、`operationTime`

### 5.4 空态与异常态
- 无工单：展示“暂无报修工单”。
- 非法流转（`1005`）：弹窗提示并刷新详情数据。

## 6. 用户/角色/权限模块

### 6.1 用户管理页
- 路由：`/system/users`
- 查询接口：`GET /api/system/users`
- 查询条件：`keyword`、`status`、`orgId`、`current`、`size`
- 列表列：`username`、`nickname`、`phone`、`email`、`status`、`orgName`、`roleCodes`、`createTime`
- 操作：
  - 新增（`sys:user:create`）
  - 编辑（`sys:user:update`）
  - 状态切换（`sys:user:update`）
  - 删除（`sys:user:delete`）
  - 分配角色（`sys:user:assign-role`）

- 新增/编辑表单字段：

| 字段 | 新增 | 编辑 | 校验 |
|---|---|---|---|
| `username` | 显示 | 只读 | 4-32 位，非空 |
| `password` | 显示 | 隐藏 | 8-64 位，非空（新增） |
| `nickname` | 显示 | 显示 | 非空 |
| `phone` | 显示 | 显示 | 可空 |
| `email` | 显示 | 显示 | 可空 |
| `orgId` | 显示 | 显示 | 非空 |
| `status` | 显示 | 显示 | 非空 |
| `roleIds` | 显示 | 显示 | 非空集合 |

### 6.2 角色管理页
- 路由：`/system/roles`
- 接口：
  - `GET /api/system/roles`
  - `GET /api/system/roles/{id}`

### 6.3 权限管理页
- 路由：`/system/permissions`

## 10. 公共组件规格

### 10.1 Markdown 编辑器组件
- 输入：`v-model=content`
- 输出：`content`（Markdown原文）
- 扩展：
  - 支持预览窗展示后端返回的 `contentHtml`。
  - 支持插入上传文件链接（文件服务返回 `accessUrl`）。

### 10.2 文件上传组件
- 上传接口：`POST /api/files/upload`
- 入参：`file`、`bizType`、`bizId`
- 返回：`FileInfoVO`
- 统一行为：
  - 上传成功后缓存 `fileId`。
  - 支持删除草稿附件（仅前端临时移除引用，不删物理文件）。
  - 下载通过 `GET /api/files/{id}/download`。

## 11. 角色独立看板页规格（阶段七-A）

### 11.1 超级管理员看板
- 路由：`/dashboard/super-admin`
- 核心卡片：用户总数、角色总数、公告发布数、活动发布数、工单总量。
- 快捷入口：用户管理、角色管理、组织管理、存储配置。

### 11.2 街道管理员看板
- 路由：`/dashboard/street-admin`
- 核心卡片：街道公告数、街道活动数、待处理工单数、社区覆盖数。
- 快捷入口：公告列表、活动列表、报修管理、组织树。

### 11.3 社区管理员看板
- 路由：`/dashboard/community-admin`
- 核心卡片：社区公告数、活动报名数、待受理工单、已完结工单。
- 快捷入口：公告列表、活动列表、报修管理。

### 11.4 物业管理员看板
- 路由：`/dashboard/property-admin`
- 核心卡片：待分派工单、处理中工单、超时工单、满意度评分。
- 快捷入口：报修管理、公告列表、活动列表、小区物业关联。

### 11.5 维修员看板
- 路由：`/dashboard/maintainer`
- 核心卡片：我的待接单、我的处理中、今日完成、催单提醒。
- 快捷入口：我的工单、工单处理、工单日志。

### 11.6 通用约束
- 看板入口必须由 `Dashboard Resolver` 计算得出，不允许前端硬编码“固定首页”。
- 多角色用户仅渲染一个主看板，按策略优先级最高项确定。
- 看板卡片仅展示当前角色有权限访问的数据与入口。
