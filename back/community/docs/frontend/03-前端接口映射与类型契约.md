# 03-前端接口映射与类型契约

## 1. 统一 HTTP 契约

### 1.1 返回结构
```ts

  code: number

  records: T[]
```

### 1.2.1 导航解析规则（阶段七-A）
- 输入：`roleCodes + permissionCodes`。
- 输出：`ResolvedNavigation`，用于动态注册路由与菜单。
- 解析优先级：`显式角色策略 > 权限签名策略 > 默认兜底策略`。
- 多角色冲突：主看板按最高优先级策略确定，菜单按权限并集构建。

### 1.3 错误码消费约定
| code | 含义 | 前端处理 |
|---|---|---|
| `0` | 成功 | 正常渲染 |
| `400` | 参数错误 | 表单提示 + 保留输入 |
| `401` | 未登录或登录失效 | 清 token，跳登录页 |
| `403` | 无权限 | 跳转 403 或提示无权限 |
| `404` | 资源不存在 | 展示空页/返回列表 |
| `409` | 数据冲突 | 保留页面并提示冲突原因 |
| `1000` | 业务异常 | 弹窗提示 message |
| `1004` | 数据范围越权 | 提示越权并回退 |
| `1005` | 非法状态流转 | 刷新详情并提示 |
| `500` | 系统异常 | 统一错误页 + traceId |

## 5. 表单字段与 DTO 约束映射
- 公告：
  - `scopeItems` 必填且非空。
  - `title/content/noticeType` 均必填。
- 活动：
  - `maxParticipants` 必填且大于 0。
  - `activityStartTime/activityEndTime/signupStartTime/signupEndTime` 均必填。
- 报修：
  - 创建时 `title/description/contactPhone/repairAddress/emergencyLevel/complexOrgId` 必填。
  - 分派时 `maintainerUserId` 必填。
  - 处理、提交、重开等动作需对应说明字段必填。
- 用户：
  - 新增 `username(4-32)/password(8-64)/nickname/orgId/status/roleIds` 必填。
- 角色：
  - `roleCode/roleName/allowCreateChild/status/permissionIds/scopeConfigs` 必填。
- 权限：
  - `permissionCode/permissionName/permissionType` 必填（新增）。
- 组织：
  - 新增 `orgCode/orgName/orgType/status/sort` 必填。

## 6. 附件与 Markdown 前端消费规范
- 所有公告与活动详情优先渲染后端返回的 `contentHtml`。
- 同时保留 `content` 供“编辑回填”和“Markdown源码查看”。
- `attachments` 为前端主消费字段；`attachmentJson` 仅用于兼容旧数据。
- 上传后返回的 `FileInfoVO.id` 作为业务关联主键：
  - 公告/活动：写入 `attachmentJson`。
  - 报修：写入 `attachmentFileIds`。
