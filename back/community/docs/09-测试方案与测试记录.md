# 09 测试方案与测试记录

## 1. 测试范围（阶段1-2）
- 登录认证：成功、失败、禁用用户
- Token 鉴权：携带/不携带/黑名单 token
- 权限拦截：无权限访问接口应被拦截
- 数据范围：跨组织写操作应拒绝
- 组织管理：编码唯一校验、逻辑删除过滤

## 2. 推荐测试方式
- 单元测试：Service 规则校验
- 接口测试：MockMvc 验证认证与权限拦截
- 手工联调：通过 Knife4j 验证核心接口

## 3. 手工测试用例示例
| 编号 | 用例 | 步骤 | 期望 |
|---|---|---|---|
| T01 | 管理员登录成功 | 使用 admin/Admin@123456 调用登录 | 返回 token |
| T02 | 登录失败 | 使用错误密码登录 | 返回登录失败错误码 |
| T03 | 无 token 访问用户列表 | 直接请求 `/api/system/users` | 返回未登录错误 |
| T04 | 越权组织写入 | 社区管理员操作其他社区组织 | 返回数据范围越权 |
| T05 | 角色越权授权 | 非超级管理员下发超权限角色 | 返回权限越权 |
