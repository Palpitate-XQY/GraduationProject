# 10 开发里程碑与提交说明

## 1. 阶段里程碑
- 阶段1：完成工程骨架、基础配置、统一响应、异常处理、安全框架、文档骨架。
- 阶段2：完成认证授权、用户角色权限、组织模型、数据范围、日志审计、SQL第一版。

## 2. 提交策略
- 按模块或阶段拆分原子提交
- 使用 Conventional Commits
- 每次提交都能独立说明一个清晰增量

## 3. 已规划提交清单
1. `chore(repo): 调整为单仓库管理并初始化后端工程骨架`
2. `build(back): 升级 Spring Boot 3 依赖与多环境配置`
3. `feat(common): 统一返回体、异常处理、错误码与traceId`
4. `feat(security): 集成 Spring Security + JWT 基础认证框架`
5. `docs(back): 初始化 01-10 技术文档骨架与架构图`
6. `feat(system): 实现用户/角色/权限与授权分配能力`
7. `feat(org): 实现组织架构与小区-物业服务关联管理`
8. `feat(scope): 实现数据范围模型与查询过滤拦截`
9. `feat(sql): 提交数据库第一版 schema/index/init 脚本`
10. `test(system): 增加认证、鉴权、数据范围核心测试与测试说明`

## 4. 风险与回滚点
- JDK版本风险：本机默认 JDK8，需切换 JDK17 再做完整编译测试。
- 数据初始化风险：若重复执行初始化 SQL，需先检查是否已存在自定义业务数据。
- 权限配置风险：新增权限点后需同步更新初始化权限与角色绑定。
