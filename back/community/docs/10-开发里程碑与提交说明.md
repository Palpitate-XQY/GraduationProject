# 10-开发里程碑与提交说明

## 1. 阶段里程碑
- 阶段1：项目骨架、基础配置、统一返回、异常处理、安全框架、文档骨架。
- 阶段2：认证授权、用户角色权限、组织管理、数据范围、基础 SQL。
- 阶段3：公告与活动模块、活动报名并发控制、SQL 扩展与文档更新。
- 阶段4：报修模块全流程、状态机、附件与流转日志、越权校验。
- 阶段5：日志查询接口、测试与部署文档收尾、索引与初始化数据优化。
- 阶段6：全流程联调资产固化（Postman 集合与环境）、联调测试补强、主链路回归验证。
- 阶段7：管理端前端功能设计与联调规范固化（仅文档交付，不新增后端模块）。

## 2. 提交策略
- 按模块拆分原子提交。
- 提交信息遵循 Conventional Commits。
- 每次提交保证可回滚、可读、可验证。

## 4. 验收基线
- 编译通过：`mvn -DskipTests compile`
- 单测通过：`mvn test`
- SQL 脚本完整：`01_schema.sql / 02_index.sql / 03_init_data.sql / 04_upgrade_notes.md`
- 文档齐全：`docs/01-10` 可用于答辩演示与后续开发交接。
- 联调资产完整：`docs/postman` 可用于端到端回归。
- 前端设计包完整：`docs/frontend` 可直接驱动前端工程落地。

## 5. 风险与后续建议
- 日志查询已支持分页与过滤，后续可补充导出与字段脱敏策略。
- 报修与操作日志随着数据增长需要归档策略。
- 公告、活动、报修可进一步增加统计分析能力与看板接口。
- 前端下一阶段建议按 `docs/frontend` 落地 Vue3 工程，并先完成管理端主链路页面，再补居民端。
