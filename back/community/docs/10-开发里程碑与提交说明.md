# 10-开发里程碑与提交说明

## 1. 阶段里程碑
- 阶段1（已完成）：
- Spring Boot 3 工程骨架、统一响应、统一异常、安全基础框架、文档骨架。
- 阶段2（已完成）：
- 认证授权、用户角色权限、组织管理、数据范围、日志基础、SQL 第一版。
- 阶段3（本次完成）：
- 公告模块（管理端 + 居民端 + 可见范围）。
- 活动模块（管理端 + 居民端 + 报名取消 + 统计 + 并发控制）。
- SQL 扩展到公告/活动模型与初始化样例数据。
- 文档更新到阶段3。

## 2. 提交策略
- 按模块/阶段拆分原子提交。
- 每个提交保证独立可读、可回滚。
- 提交信息遵循 Conventional Commits。

## 3. 已完成提交（阶段1-2）
1. `chore(repo): 调整为单仓库管理并初始化后端工程骨架`
2. `build(back): 升级 Spring Boot 3 依赖与多环境配置`
3. `feat(common): 统一返回体、异常处理、错误码与traceId`
4. `feat(security): 集成 Spring Security + JWT 基础认证框架`
5. `docs(back): 初始化 01-10 技术文档骨架与架构图`
6. `feat(system): 实现用户/角色/权限与授权分配能力`
7. `feat(org): 实现组织架构与小区-物业服务关联管理`
8. `feat(scope): 实现数据范围模型与查询过滤拦截`
9. `feat(log): 增加操作日志 AOP 与持久化`
10. `feat(sql): 提交数据库第一版 schema/index/init 脚本`
11. `test(system): 增加认证、鉴权、数据范围核心测试与测试说明`
12. `chore(config): 保留本地构建与开发环境配置调整`

## 4. 阶段3提交计划（本次执行）
1. `feat(notice): 实现公告模块 CRUD、发布撤回与可见范围`
2. `feat(activity): 实现活动模块 CRUD、报名流程与并发控制`
3. `feat(sql): 扩展公告活动表结构、索引与初始化数据`
4. `docs(back): 更新阶段三接口与测试文档`

## 5. 验收基线
- 代码编译：`mvn -DskipTests compile` 通过。
- 单元测试：`mvn test` 通过。
- 接口可通过 SpringDoc/Knife4j 在线查看并联调。
- SQL 脚本可由人工按 `01 -> 02 -> 03` 顺序执行。

## 6. 风险与回滚点
- 若活动报名并发策略需切换，可仅回滚 `ActivitySignupLimiter` 与 `ActivityServiceImpl` 相关提交。
- 若数据范围策略要收紧，可回滚 `DataScopeServiceImpl` 与相关 Mapper 调整。
- SQL 扩表与初始化数据拆分提交，便于按模块独立回退。
