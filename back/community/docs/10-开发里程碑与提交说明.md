# 10-开发里程碑与提交说明

## 1. 阶段里程碑
- 阶段1（已完成）：
- Spring Boot 3 工程骨架、统一响应、统一异常、安全基础框架、文档骨架。
- 阶段2（已完成）：
- 认证授权、用户角色权限、组织管理、数据范围、日志基础、SQL 第一版。
- 阶段3（已完成）：
- 公告模块、活动模块、活动报名并发控制、SQL 与文档同步更新。
- 阶段4（本次完成）：
- 报修模块全流程、状态机校验、流转日志、附件关联、权限与越权拦截。
- 报修相关 SQL 扩展、初始化权限与示例数据、测试与文档更新。

## 2. 提交策略
- 按模块拆分原子提交，确保每次提交可读、可回滚。
- 提交信息遵循 Conventional Commits。

## 3. 阶段1-3既有提交（节选）
1. `feat(notice): 实现公告模块 CRUD、发布撤回与可见范围`
2. `feat(activity): 实现活动模块 CRUD、报名流程与并发控制`
3. `feat(scope): 扩展物业公司服务小区数据范围过滤`
4. `feat(sql): 扩展公告活动表结构、索引与初始化数据`
5. `docs(back): 更新阶段三接口与测试文档`

## 4. 阶段4提交计划（本次执行）
1. `feat(repair): 实现报修工单全流程与状态机校验`
2. `feat(sql): 扩展报修表结构、索引与初始化权限数据`
3. `docs(back): 更新阶段四报修接口、流程与测试文档`
4. `test(repair): 增加报修状态机核心规则测试`

## 5. 阶段4验收基线
- 代码编译：`mvn -DskipTests compile` 通过。
- 自动化测试：`mvn test` 通过（总计 6 项）。
- 报修接口可通过 Swagger/Knife4j 查看并联调。
- SQL 脚本按 `01_schema.sql -> 02_index.sql -> 03_init_data.sql` 顺序手动执行。

## 6. 风险与回滚点
- 如果报修流转规则需调整，优先回滚 `RepairStateMachine` 与 `RepairOrderServiceImpl`。
- 如果报修查询性能不足，优先调整 `biz_repair_order` 组合索引而非改业务字段。
- SQL 初始化权限采用追加写法，可按权限段落独立回滚。
