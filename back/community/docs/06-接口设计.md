|---|---|---|---|
| `/api/auth/login` | POST | 无 | 登录获取 Token |

## 8. 公告/活动/报修/日志接口
- 公告：管理端与居民端双视图（含发布/撤回、可见范围）。
- 活动：管理端 CRUD + 居民报名/取消 + 报名统计。
- 报修：居民创建与反馈、物业受理分派、维修处理、状态流转日志。
- 日志：`GET /api/logs/logins`、`GET /api/logs/operations`。

## 10. Markdown 内容约定
- 公告与活动的 `content` 字段统一按 Markdown 存储。
- 查询接口在返回 `content` 的同时新增 `contentHtml` 字段（后端渲染后的 HTML）。
- 查询接口在保留 `attachmentJson` 的同时新增结构化 `attachments` 字段，字段包含 `fileId/originFileName/fileName/fileSize/contentType/storageType/accessUrl`。
- 前端可按需直接展示 `contentHtml`，也可自行渲染 `content`。

### 10.1 联调字段说明
| 字段 | 所在接口 | 类型 | 说明 |
|---|---|---|---|
| `content` | 公告/活动 列表与详情 | string | Markdown 原文，前端可二次渲染 |
| `contentHtml` | 公告/活动 列表与详情 | string | 后端渲染后的 HTML，前端可直接展示 |
| `attachmentJson` | 公告/活动 列表与详情 | string | 附件引用原始 JSON（保持兼容） |
| `attachments` | 公告/活动 列表与详情 | array | 结构化附件列表，建议前端优先使用 |

### 10.2 公告详情响应示例
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "id": 8,
    "title": "stage6_notice_260324185509",
    "content": "# Stage6 Notice\n- markdown\n- attachment",
    "contentHtml": "<h1>Stage6 Notice</h1>\n<ul>\n<li>markdown</li>\n<li>attachment</li>\n</ul>\n",
    "attachmentJson": "{\"attachments\":[{\"fileId\":13}]}",
    "attachments": [
      {
        "fileId": 13,
        "originFileName": "notice-stage6.md",
        "fileName": "xxxxxx.md",
        "fileSize": 54,
        "contentType": "application/octet-stream",
        "storageType": "LOCAL",
        "accessUrl": "/api/files/13/download"
      }
    ]
  },
  "traceId": "demo-trace-id"
}
```

## 11. 核心状态值
- 公告：`DRAFT/PUBLISHED/RECALLED`
- 活动：`DRAFT/PUBLISHED/RECALLED/FINISHED`
- 报修：`PENDING_ACCEPT/ACCEPTED/ASSIGNED/PROCESSING/WAIT_CONFIRM/DONE/CLOSED/REJECTED/REOPENED`

## 12. 通用错误码（示例）
- `UNAUTHORIZED`：未登录或 Token 失效。
- `FORBIDDEN`：无权限访问。
- `VALIDATE_FAILED`：参数校验失败。
- `ILLEGAL_STATE_FLOW`：非法状态流转。
- `BIZ_ERROR`：业务异常。
