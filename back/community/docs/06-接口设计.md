# 06-接口设计

## 1. 统一返回结构
```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "traceId": "f3577fe3ee4b4f9b"
}
```

## 2. 分页结构
```json
{
  "records": [],
  "total": 0,
  "current": 1,
  "size": 10
}
```

## 3. 认证接口
| 接口 | 方法 | 权限 | 说明 |
|---|---|---|---|
| `/api/auth/login` | POST | 无 | 登录获取 Token |
| `/api/auth/resident/register` | POST | 无 | 居民自助注册 |
| `/api/auth/password/reset-code` | POST | 无 | 忘记密码发送验证码 |
| `/api/auth/password/reset-by-code` | POST | 无 | 验证码重置密码 |
| `/api/auth/logout` | POST | 已登录 | 登出并拉黑 Token |
| `/api/auth/me` | GET | 已登录 | 获取当前用户信息 |
| `/api/auth/password` | PUT | 已登录 | 已登录用户修改密码 |

## 4. 居民注册请求参数
| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `username` | string | 是 | 用户名（4-32位） |
| `password` | string | 是 | 登录密码（8-64位） |
| `nickname` | string | 是 | 昵称 |
| `phone` | string | 是 | 手机号 |
| `email` | string | 否 | 邮箱 |
| `complexOrgId` | long | 是 | 所属小区ID |
| `roomNo` | string | 是 | 房号 |
| `emergencyContact` | string | 否 | 紧急联系人 |
| `emergencyPhone` | string | 否 | 紧急联系电话 |

## 5. 忘记密码接口参数
### 5.1 发送验证码 `/api/auth/password/reset-code`
| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `username` | string | 是 | 用户名 |
| `phone` | string | 是 | 绑定手机号 |

### 5.2 验证码重置 `/api/auth/password/reset-by-code`
| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| `username` | string | 是 | 用户名 |
| `phone` | string | 是 | 绑定手机号 |
| `code` | string | 是 | 6位验证码 |
| `newPassword` | string | 是 | 新密码（8-64位） |

## 6. 系统管理接口
| 分组 | 主要接口 | 权限前缀 |
|---|---|---|
| 用户 | `/api/system/users`、`/api/system/users/{id}`、`/api/system/users/roles` | `sys:user:*` |
| 角色 | `/api/system/roles`、`/api/system/roles/{id}` | `sys:role:*` |
| 权限 | `/api/system/permissions`、`/api/system/permissions/{id}` | `sys:permission:*` |

## 8. 公告/活动/报修/日志接口
- 公告：管理端与居民端双视图（含发布/撤回、可见范围）。
- 活动：管理端 CRUD + 居民报名/取消 + 报名统计。
- 报修：居民创建与反馈、物业受理分派、维修处理、状态流转日志。
- 日志：`GET /api/logs/logins`、`GET /api/logs/operations`。

## 9. 文件与存储配置接口
| 接口 | 方法 | 权限 | 说明 |
|---|---|---|---|
| `/api/files/upload` | POST | `file:upload` | 上传附件，支持活动图片/活动文件/报修图片等 |
| `/api/files/{id}` | GET | `file:view` | 查询文件元信息与访问地址 |
| `/api/files/{id}/download` | GET | `file:view` | 下载文件（本地直出，七牛302跳转） |
| `/api/system/storage-config` | GET | `sys:storage:view` | 查询当前存储配置（仅超级管理员） |
| `/api/system/storage-config` | PUT | `sys:storage:update` | 更新存储配置，切换 LOCAL/QINIU（仅超级管理员） |

## 10. Markdown 内容约定
- 公告与活动的 `content` 字段统一按 Markdown 存储。
- 查询接口在返回 `content` 的同时新增 `contentHtml` 字段（后端渲染后的 HTML）。
- 查询接口在保留 `attachmentJson` 的同时新增结构化 `attachments` 字段，字段包含 `fileId/originFileName/fileName/fileSize/contentType/storageType/accessUrl`。
- 前端可按需直接展示 `contentHtml`，也可自行渲染 `content`。

## 11. 核心状态值
- 公告：`DRAFT/PUBLISHED/RECALLED`
- 活动：`DRAFT/PUBLISHED/RECALLED/FINISHED`
- 报修：`PENDING_ACCEPT/ACCEPTED/ASSIGNED/PROCESSING/WAIT_CONFIRM/DONE/CLOSED/REJECTED/REOPENED`

## 12. 通用错误码（示例）
- `UNAUTHORIZED`：未登录或 Token 失效。
- `FORBIDDEN`：无权限访问。
- `VALIDATE_FAILED`：参数校验失败。
- `ILLEGAL_STATE_FLOW`：非法状态流转。
- `BIZ_ERROR`：业务异常。
