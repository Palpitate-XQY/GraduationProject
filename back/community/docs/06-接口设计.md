# 06 接口设计

## 1. 统一返回结构
```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "traceId": "xxxxxx"
}
```

## 2. 分页结构
```json
{
  "records": [],
  "total": 0,
  "current": 1,
  "size": 10
}
```

## 3. 认证接口
- `POST /api/auth/login` 登录
- `POST /api/auth/logout` 退出登录
- `GET /api/auth/me` 当前用户信息
- `PUT /api/auth/password` 修改密码

## 4. 用户/角色/权限接口
- 用户：
  - `GET /api/system/users`
  - `GET /api/system/users/{id}`
  - `POST /api/system/users`
  - `PUT /api/system/users`
  - `DELETE /api/system/users/{id}`
  - `PUT /api/system/users/{id}/status`
  - `PUT /api/system/users/roles`
- 角色：
  - `GET /api/system/roles`
  - `GET /api/system/roles/{id}`
  - `POST /api/system/roles`
  - `PUT /api/system/roles`
  - `DELETE /api/system/roles/{id}`
- 权限：
  - `GET /api/system/permissions`
  - `GET /api/system/permissions/{id}`
  - `POST /api/system/permissions`
  - `PUT /api/system/permissions`
  - `DELETE /api/system/permissions/{id}`

## 5. 组织接口
- `GET /api/org/tree` 组织树
- `POST /api/org` 新增组织
- `PUT /api/org` 编辑组织
- `DELETE /api/org/{id}` 删除组织
- `GET /api/org/complex-property-rel` 查询小区物业关联
- `POST /api/org/complex-property-rel` 新增小区物业关联
- `DELETE /api/org/complex-property-rel/{id}` 删除小区物业关联

## 6. 权限要求示例
- `sys:user:list`：查询用户分页
- `sys:role:create`：新增角色
- `org:create`：新增组织
- `org:complex-property:create`：新增小区物业关联

## 7. 状态值说明（阶段1-2）
- 用户状态：`1=启用,0=禁用`
- 角色状态：`1=启用,0=禁用`
- 组织状态：`1=启用,0=禁用`
- 登录日志：`success_flag 1=成功,0=失败`
