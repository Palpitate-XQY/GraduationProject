# 06-接口设计

## 1. 统一返回结构
```json
{
  "code": 0,
  "message": "success",
  "data": {},
  "traceId": "f3577fe3ee4b4f9b"
}
```

## 2. 分页返回结构
```json
{
  "records": [],
  "total": 0,
  "current": 1,
  "size": 10
}
```

## 3. 认证接口
- `POST /api/auth/login`：账号密码登录，返回 `token`、过期时间、用户信息。
- `POST /api/auth/logout`：退出登录，写入 Redis 黑名单。
- `GET /api/auth/me`：获取当前登录用户信息与权限点集合。
- `PUT /api/auth/password`：修改当前用户密码。

## 4. 用户/角色/权限接口（阶段2）
- 用户管理：
- `GET /api/system/users`
- `GET /api/system/users/{id}`
- `POST /api/system/users`
- `PUT /api/system/users`
- `DELETE /api/system/users/{id}`
- `PUT /api/system/users/{id}/status`
- `PUT /api/system/users/roles`
- 角色管理：
- `GET /api/system/roles`
- `GET /api/system/roles/{id}`
- `POST /api/system/roles`
- `PUT /api/system/roles`
- `DELETE /api/system/roles/{id}`
- `PUT /api/system/roles/permissions`
- `PUT /api/system/roles/scopes`
- 权限管理：
- `GET /api/system/permissions`
- `GET /api/system/permissions/{id}`
- `POST /api/system/permissions`
- `PUT /api/system/permissions`
- `DELETE /api/system/permissions/{id}`

## 5. 组织管理接口（阶段2）
- `GET /api/org/tree`：组织树查询（支持关键字、类型、状态过滤）。
- `POST /api/org`：新增组织（`orgCode` 唯一）。
- `PUT /api/org`：更新组织。
- `DELETE /api/org/{id}`：逻辑删除组织。
- `GET /api/org/complex-property-rel`：查询小区-物业服务关联。
- `POST /api/org/complex-property-rel`：新增小区-物业服务关联。
- `DELETE /api/org/complex-property-rel/{id}`：删除小区-物业服务关联。

## 6. 公告接口（阶段3）
- 管理端：
- `GET /api/notices`
- `GET /api/notices/{id}`
- `POST /api/notices`
- `PUT /api/notices`
- `DELETE /api/notices/{id}`
- `POST /api/notices/{id}/publish`
- `POST /api/notices/{id}/recall`
- 居民端：
- `GET /api/notices/resident/page`
- `GET /api/notices/resident/{id}`

### 公告请求说明
- 创建/更新请求包含：`noticeType`、`title`、`content`、`topFlag`、`scopeItems`。
- `scopeItems` 为可见范围集合，元素结构：`scopeType + scopeRefId`。
- 物业公告限制：`noticeType=PROPERTY` 时仅允许投放到小区范围，且必须属于物业公司服务关系。

## 7. 活动接口（阶段3）
- 管理端：
- `GET /api/activities`
- `GET /api/activities/{id}`
- `POST /api/activities`
- `PUT /api/activities`
- `DELETE /api/activities/{id}`
- `POST /api/activities/{id}/publish`
- `POST /api/activities/{id}/recall`
- `GET /api/activities/{id}/signups`
- `GET /api/activities/{id}/stats`
- 居民端：
- `GET /api/activities/resident/page`
- `GET /api/activities/resident/{id}`
- `POST /api/activities/{id}/signup`
- `POST /api/activities/{id}/cancel-signup`

### 活动请求说明
- 创建/更新请求包含：
- `title`、`content`、`activityStartTime`、`activityEndTime`
- `signupStartTime`、`signupEndTime`
- `location`、`maxParticipants`
- `scopeItems`（可见范围）
- 报名并发控制：`signup` 接口通过 Redis Lua 做名额预占，数据库唯一索引防重复报名。

## 8. 权限点示例
- `sys:user:list`：查询用户分页。
- `sys:role:create`：新增角色。
- `org:create`：新增组织。
- `notice:publish`：发布公告。
- `activity:signup`：居民报名活动。

## 9. 关键状态值
- 用户状态：`1=启用`，`0=禁用`。
- 角色状态：`1=启用`，`0=禁用`。
- 组织状态：`1=启用`，`0=禁用`。
- 公告状态：`DRAFT=草稿`，`PUBLISHED=已发布`，`RECALLED=已撤回`。
- 活动状态：`DRAFT=草稿`，`PUBLISHED=已发布`，`RECALLED=已撤回`，`FINISHED=已结束`。
- 报名状态：`SIGNED=已报名`，`CANCELED=已取消`。

## 10. 鉴权与数据范围要求
- 所有管理端接口统一走 Spring Security 鉴权。
- 先验权限：`@PreAuthorize("hasAuthority('xxx')")`。
- 后验数据范围：Service 层统一调用 `DataScopeService` 校验组织可见性。
- 越权访问返回统一错误码 `DATA_SCOPE_DENIED` 或 `FORBIDDEN`。
