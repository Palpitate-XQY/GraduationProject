# 08-部署运行说明

## 1. 环境要求
- JDK 17+
- Maven 3.9+
- MySQL 8.x
- Redis 6.x+

## 2. 目录说明
- 后端工程：`back/community`
- SQL 脚本：`back/community/sql`
- 技术文档：`back/community/docs`

## 3. 数据库初始化（手动执行）
在 MySQL 中创建数据库（建议名：`community_service`），并按顺序执行：
1. `sql/01_schema.sql`
2. `sql/02_index.sql`
3. `sql/03_init_data.sql`

说明：
- 项目不强依赖 Flyway/Liquibase。
- 采用手工 SQL 初始化，便于毕业设计答辩演示与版本回溯。

## 4. 配置文件
- 主配置：`src/main/resources/application.yml`
- 开发环境：`src/main/resources/application-dev.yml`
- 生产环境：`src/main/resources/application-prod.yml`
- 本地示例：`src/main/resources/application-local.yml.example`

建议通过环境变量注入敏感配置：
- `COMMUNITY_DB_URL`
- `COMMUNITY_DB_USERNAME`
- `COMMUNITY_DB_PASSWORD`
- `COMMUNITY_REDIS_HOST`
- `COMMUNITY_REDIS_PORT`
- `COMMUNITY_REDIS_PASSWORD`
- `COMMUNITY_JWT_SECRET`

## 5. 本地启动
```bash
mvn clean spring-boot:run -Dspring-boot.run.profiles=dev
```

默认访问：
- 服务地址：`http://localhost:8080`
- Swagger UI：`http://localhost:8080/swagger-ui/index.html`
- Knife4j：`http://localhost:8080/doc.html`

## 6. 默认管理员
- 用户名：`admin`
- 初始密码：`Admin@123456`
- 首次登录强制改密：`must_change_password=1`

## 7. 本地联调建议顺序
1. 执行 SQL 脚本并确认表、索引、初始数据存在。
2. 启动 Redis。
3. 启动应用并用管理员登录。
4. 在接口文档中依次验证：认证、系统管理、组织、公告、活动、报修、日志查询。

## 8. 常见问题
- MySQL 连接失败：检查数据库地址、用户名、密码、字符集和时区参数。
- Redis 连接失败：检查服务是否启动、密码与端口是否正确。
- 接口返回 401：确认携带 `Authorization: Bearer <token>`。
- 接口返回 403：确认当前角色已授予对应权限点。
- 登录后提示必须改密：属于系统预置管理员安全策略，改密后可继续操作。
